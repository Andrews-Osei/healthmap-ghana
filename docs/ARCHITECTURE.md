# HealthMap Ghana — Architecture

## Overview

HealthMap Ghana is a three-tier system: a Next.js + TypeScript frontend, a
FastAPI service, and a data layer that begins life as pre-computed JSON and
upgrades to PostgreSQL + PostGIS in v2. The ML pipeline is an offline batch
that produces the JSON artefacts the API serves.

```
+----------------+      HTTPS / JSON       +------------------+
|  Next.js 14    | <---------------------> |  FastAPI         |
|  TypeScript    |                         |  Pydantic v2     |
|  Tailwind      |                         |  CORS + JWT      |
|  Leaflet       |                         |                  |
|  Recharts      |                         +--------+---------+
|  Framer Motion |                                  |
+----------------+                                  |
                                                    v
                                       +-----------------------+
                                       |  data layer (v1)      |
                                       |  /backend/app/data/   |
                                       |  *.json (in-mem cache)|
                                       +-----------+-----------+
                                                   |
                                                   |  upgrade
                                                   v
                                       +-----------------------+
                                       |  PostgreSQL + PostGIS |
                                       |  (db/schema.sql)      |
                                       +-----------------------+

                       Offline batch
+--------------------------------------------------+
|  ML pipeline  (ml/pipeline/)                     |
|  ETL  ->  features  ->  scoring  ->  forecast    |
|                          |        ->  recs       |
|  outputs: districts.json regions.json            |
|           facilities.json recommendations.json   |
|           forecast.json   manifest.json          |
+--------------------------------------------------+
```

## Frontend

Next.js 14 app router, React 18, TypeScript, Tailwind CSS. State in Zustand.
Map: Leaflet via `react-leaflet` (Mapbox GL is a drop-in alternative). Charts:
Recharts. Animations: Framer Motion. Icons: Lucide.

Routes:
- `/`          — landing page
- `/dashboard` — interactive analytics

Key components:
- `Hero`, `StatsCards`, `FeatureGrid`, `HowItWorks`, `DashboardPreview`,
  `WhyItMatters`, `CTA` — landing
- `ChoroplethMap`, `FilterPanel`, `DistrictSidebar`, `TopUnderservedTable`,
  `RegionBreakdown`, `RecommendationsCard`, `ForecastChart`,
  `VulnerabilityGauge`, `KpiStrip` — dashboard

Design tokens (see `tailwind.config.ts`): `ink` (dark blue), `cyan`,
`med` (healthcare green), `accent`. Glassmorphism via `.glass` /
`.glass-strong`. Skeleton shimmers via `.skeleton`.

## Backend

FastAPI 0.110+. All routes under `/api/v1`. CORS configured from env.
Routers: `districts`, `regions`, `facilities`, `vulnerability`,
`recommendations`, `forecast`, `auth`, `health`.

JWT auth scaffold (`services/security.py`) with bcrypt and OAuth2 password
flow. Demo users: `demo` / `demo`, `admin` / `admin`.

In v1, all endpoints read JSON via `services/data_loader.py` (LRU-cached).
In v2, swap each loader for a SQLAlchemy query — the public function
signatures stay identical.

## Data layer

**v1 (current):** JSON artefacts in `backend/app/data/`. Generated by the
ML pipeline. Atomic to ship, zero-DB to run.

**v2 (planned):** PostgreSQL 14+ with PostGIS 3+. Schema in `db/schema.sql`.
ETL DAG to refresh `district_metrics`, `recommendations`, `forecasts`
nightly. Add Alembic migrations.

## ML pipeline

Six stages, all pure Python (NumPy / Pandas, optional scikit-learn):

1. **ETL** — read GeoPackage, decode geometries, normalise types
2. **Features** — per-district aggregations, per-capita normalisation,
   urban/rural classifier
3. **Scoring** — Composite Vulnerability Index (CVI), 0-100
4. **Clustering** — K-Means (k=4), tier names ordered by mean CVI
5. **Forecasting** — population growth → required-vs-current shortfall
6. **Recommendations** — rule-based intervention engine, ranked

See [ML_PIPELINE.md](ML_PIPELINE.md) for formulas.

## Deployment topology

Recommended: containerised. `docker-compose.yml` brings up `db`, `backend`,
`frontend`. Production target: Kubernetes with the same three Deployments
plus a ReadOnlyMany volume mount for the JSON artefacts (or full DB
migration).

See [DEPLOYMENT.md](DEPLOYMENT.md) for step-by-step.

## Future integrations (designed-for, not implemented)

- **Road-network analysis (OSRM)** — hook in `services/`
- **Satellite imagery overlays** — Mapbox tile layer or self-hosted COG
- **Ambulance routing** — extend forecast routes with isochrone queries
- **Disaster-response overlays** — versioned `events` table (planned)
- **Disease-outbreak tracking** — time-series `epi_observations` table

The `data/` folder structure already reserves `population/` and `geo/`
subfolders for these layers.
